## 1. 这个工具是干什么的？

**周树人**是一款在**你自己电脑本地**运行的网页小工具，用来辅助写**知识科普类短视频口播稿**。

它能帮你：

| 步骤 | 你能做什么 |
|------|------------|
| 导入参考 | 把栏目里以前的 Word / 文本稿上传进来，当「风格样本」 |
| 写大纲 | 输入本期视频要讲什么（几十字即可） |
| AI 写稿 | 根据「创作者身份 + 大纲 +（可选）参考文稿」生成约 500 字逐字稿 |
| 改稿 | 用鼠标**拖选**某句话，让 AI 按你的要求改写 |
| 查证 | 对可疑句子做「信源请求」，从学术库 / 维基检索**真实链接**（需人工点开核对） |
| 定稿 | 生成多条**标题、简介**备选（强调吸引人、有趣味） |
| 导出 | 复制全文，或下载 **Word / Markdown** |

> **重要：** 工具通过你配置的 **API Key** 调用豆包 / DeepSeek / OpenRouter 等 AI 服务，**按厂商规则计费**。Key 只保存在本机浏览器，不会上传到我们的 GitHub。

## 2. 使用前需要准备什么？

| 准备项 | 说明 | 怎么获得 |
|--------|------|----------|
| **电脑** | Mac 或 Windows 均可 | — |
| **浏览器** | 推荐 Chrome、Edge、Safari 最新版 | 系统自带或官网下载 |
| **Node.js** | 用来在本机「启动」这个网页工具（不是写代码） | 打开 https://nodejs.org/ 下载 **LTS（长期支持版）**，一路「下一步」安装 |
| **终端** | 一个黑色或白色的「命令行窗口」，用来输入启动命令 | Mac：「启动台 → 终端」；Windows：开始菜单搜 **PowerShell** 或 **命令提示符** |
| **Git（可选）** | 用来从 GitHub 拉取最新代码 | 不用 Git 也可以：在 GitHub 网页点 **Code → Download ZIP** 解压 |
| **API Key** | 调用 AI 的钥匙，**每人单独申请** | 见 [第 5 节](#5-配置-ai-密钥api-key) |
| **网络** | 生成脚本、检索信源时需要联网 | 国内用豆包 / DeepSeek 一般即可；OpenRouter 可能需要科学上网 |

## 3. 第一次安装（只需做一次）

### 3.1 获取项目文件

**方式 A：Git 克隆（推荐，方便以后更新）**

在终端里**逐行**复制粘贴（每行回车一次）：

```bash
git clone https://github.com/wangyanlinbj/script-workshop.git
cd script-workshop
```

> 若仓库是**私有的**，需要管理员在 GitHub 把你加为协作者后，用你自己的 GitHub 账号登录再克隆。

**方式 B：下载 ZIP**

1. 打开 https://github.com/wangyanlinbj/script-workshop  
2. 点绿色 **Code** → **Download ZIP**  
3. 解压到例如「桌面 / script-workshop」  
4. 在终端里 `cd` 进入解压后的文件夹，例如：

```bash
cd ~/Desktop/script-workshop
```

（Windows 路径可能是 `cd C:\Users\你的用户名\Desktop\script-workshop`）

### 3.2 安装依赖

确保终端当前目录是**能看到 `package.json` 的那一层**（文件夹名可能是 `script-workshop` 或 `web`，以你解压/克隆结果为准），然后执行：

```bash
npm install
```

- 第一次可能需要 **1～5 分钟**，会下载一些依赖包，属正常现象。  
- 若出现 `npm: command not found`，说明 Node.js 没装好，请回到 [nodejs.org](https://nodejs.org/) 重装 LTS 并**重启终端**。

### 3.3 验证能否启动

```bash
npm run dev
```

成功时，终端里会出现类似：

```text
  ➜  Local:   http://localhost:5173/
```

用浏览器打开 **http://localhost:5173/**（没自动打开就手动输入地址栏）。

看到「周树人」页面即安装成功。  
**先不要关终端窗口**——关掉就等于关掉了工具。

## 4. 每次怎么打开工具？

1. 打开**终端**  
2. `cd` 进入项目目录（和安装时同一个文件夹）  
3. 执行：

```bash
npm run dev
```

4. 浏览器访问终端里显示的地址（一般是 `http://localhost:5173/`）

### 千万不要这样做

| 错误做法 | 后果 |
|----------|------|
| 直接双击 `index.html` | AI 无法调用，会报网络 / 跨域错误 |
| 关掉终端但还想用网页 | 页面会失效，需要重新 `npm run dev` |

## 5. 配置 AI 密钥（API Key）

### 5.1 打开配置窗口

1. 启动工具并打开网页  
2. 右上角点 **⚙ API Key**  
3. 先在顶部 **AI 引擎** 下拉框里选你要用的服务商  

### 5.2 填写 Key 和「模型 ID」

弹窗里每个引擎有两项：

- **API Key**：厂商给你的密钥（像密码，勿发微信群、勿截图外传）  
- **模型 ID**：具体用哪一个 AI 模型（**不是**在下拉里选版本，要你自己填一串 ID）

填好后**会自动保存在本机浏览器**。换电脑、换浏览器、清空浏览器缓存后需要**重新填**。

### 5.3 三个引擎怎么选？

| 引擎 | 适合谁 | API Key 从哪里申请 | 模型 ID 示例 |
|------|--------|-------------------|--------------|
| **豆包** | 国内同事首选，速度稳定 | [火山引擎 ARK 控制台 → API Key 管理](https://console.volcengine.com/ark/region:ark+cn-beijing/apiKey)（必须是 **ARK 专用 Key**，不是账号通用 AccessKey） | 推荐填推理接入点：`ep-xxxxxxxx-xxxxx`；或公开模型如 `doubao-1-5-pro-32k-250115` |
| **DeepSeek** | 国内、性价比高 | [platform.deepseek.com/api_keys](https://platform.deepseek.com/api_keys) | `deepseek-chat`、`deepseek-reasoner` |
| **OpenRouter** | 想用 Claude、GPT 等国外模型 | [openrouter.ai/keys](https://openrouter.ai/keys) | 如 `anthropic/claude-sonnet-4.5`、`openai/gpt-4o`（完整列表见 [openrouter.ai/models](https://openrouter.ai/models)） |

### 5.4 常见 Key 报错

| 报错大意 | 处理 |
|----------|------|
| 请先填写 API Key / 模型 ID | 打开 ⚙ 弹窗，确认当前引擎两项都已填 |
| 豆包 Key 格式不对 | 不要用火山引擎「通用 AccessKey」，要用 ARK 里的 API Key |
| 豆包模型不存在 | 模型 ID 全小写、用 `-` 连接；优先用控制台里的 `ep-...` 接入点 ID |
| OpenRouter 失败 | 检查 Key 余额；本机可能需要科学上网，或改豆包 / DeepSeek |

## 6. 界面长什么样？

页面大致是 **三栏**：

```
┌──────────────┬────────────────────────────┬──────────────┐
│  左：栏目     │  中：创作区                 │  右：参考文献  │
│  + 参考文稿   │  创作者身份 / 大纲 / 脚本   │  信源检索结果  │
└──────────────┴────────────────────────────┴──────────────┘
```

| 区域 | 作用 |
|------|------|
| **顶部** | Logo、切换 AI 引擎、⚙ API Key |
| **左侧「栏目 & 参考文稿」** | 上传文件夹、展开栏目看 Word 列表、预览文稿、删除栏目 |
| **中间** | 写大纲、生成脚本、改稿、确认稿件、选标题简介 |
| **底部横条** | 选好标题+简介后出现：复制全文、导出 Word / Markdown |
| **右侧「参考文献」** | 信源请求后的检索记录，可展开查看链接 |

## 7. 准备参考文稿（上传文件夹）

### 7.1 文件夹怎么整理？

请用一个**总文件夹**，里面**每个子文件夹 = 一个栏目**，子文件夹里直接放文稿（不要套太多层）：

```text
我的参考文稿/
├── 科普实验室/          ← 栏目名
│   ├── 第01期-睡眠.docx
│   └── 第02期-记忆.docx
└── 动物世界/
    └── 样稿-深海鱼.docx
```

支持格式：**`.docx`（Word）**、**`.txt`**、**`.md`**。  
旧版 `.doc` 可能无法解析正文，建议另存为 `.docx`。

### 7.2 上传步骤

1. 左侧点 **+ 上传文件夹**  
2. 选中上面的「我的参考文稿」这一层（父文件夹）  
3. 等待「解析文档中…」结束，左侧会出现栏目名  
4. 点栏目左侧 **▸** 可展开看文稿列表；点某篇可在左下角**预览**  

同一栏目名再次上传会**合并/更新**同名文件，不会重复建栏目。

## 8. 完整创作流程（推荐顺序）

按下面顺序做，最不容易乱：

| 步骤 | 操作 | 说明 |
|:----:|------|------|
| ① | 启动 `npm run dev`，配置 API Key | 见上文 |
| ② | 上传参考文件夹，**点击选中一个栏目** | 未选栏目时中间是空白提示 |
| ③ | （可选）编辑 **创作者身份** | 定义 AI 扮演的作者人设、结构、语气；可点「恢复默认」 |
| ④ | 填写 **创作大纲**（约 50 字） | 写清本期讲什么、角度、案例 |
| ⑤ | （可选）打开 **参考资料库** 开关 | 开启后生成时会参考该栏目已导入的文稿风格 |
| ⑥ | 点 **✦ 生成脚本** | 等待流式输出完成，下方显示字数 |
| ⑦ | **拖选**需要修改的句子 | 松开后出现修改卡片，输入意见，多轮对话后点 **✓ 应用到脚本** |
| ⑧ | （可选）点 **信源请求** | 对选中句子查参考文献，见 [第 10 节](#10-参考文献--信源查证) |
| ⑨ | 满意后点 **✓ 确认稿件** | 自动生成标题、简介备选 |
| ⑩ | 各选一条 **标题** 和 **简介** | 点击某条即选中（高亮） |
| ⑪ | 底部 **导出 Word** 或 **复制全文** | 需标题、简介都已选 |

## 9. 常用功能说明

### 9.1 创作者身份

- 位置：选中栏目后，大纲上方的可折叠区域  
- 作用：告诉 AI「你是谁、怎么写、什么结构」，**每次生成脚本都会带上**  
- 修改后会自动保存到本机；点 **恢复默认** 可还原团队预设  

### 9.2 参考资料库（开关）

- 位置：大纲输入框右下角  
- **开启**：把当前栏目下的参考文稿摘要注入 AI，并模仿其语言风格  
- **关闭**：只根据创作者身份 + 大纲写稿  
- 参考文稿越多，生成可能越慢，属正常  

### 9.3 生成脚本 / 重新生成

- **✦ 生成脚本**：按当前大纲重新写一整篇（会覆盖已有脚本草稿）  
- 逐字稿卡片上的 **重新生成**：同上，写前请确认是否还要保留旧稿  

### 9.4 拖选修改（不是点段落）

1. 在脚本草稿里**按住鼠标拖选**一段文字  
2. 松开后弹出 **选中内容修改** 卡片  
3. 在下方输入修改要求，例如：「更口语」「缩短到 80 字」「加一句反问」  
4. 按 **Enter** 发送（**Shift+Enter** 换行）  
5. AI 回复后点 **✓ 应用到脚本**，修改才会写回正文  

可多次对话，直到满意再应用。

## 10. 参考文献 / 信源查证

用于核对口播稿里**事实性表述**是否有依据。

### 怎么用？

1. 先生成脚本草稿  
2. **拖选**要查证的一句话  
3. 在修改卡片右上角点 **信源请求**  
4. 右侧「参考文献」栏会显示检索进度和结果  

### 结果从哪来？

工具会：

1. 用 AI 分析句子，生成**英文检索关键词**（不直接编链接）  
2. 去 **OpenAlex、Crossref、中文维基百科** 等公开库搜索  
3. 列出可点击的链接  

### 使用时请注意

| 说明 | |
|------|-----|
| 链接是**真实检索**来的，比 AI 凭空写 URL 可靠 | 但仍需你**点开链接**，看是否支持原句 |
| 检索结果未必完全对口播句 | 尤其是中文科普表述，可能只有相关论文或百科背景 |
| 必须在 `npm run dev` 下使用 | 静态部署的网站需要技术同事配置同样的 API 代理 |

## 11. 标题与简介

点击 **✓ 确认稿件** 后，中间下方会出现两块：

| 模块 | 规则 | 操作 |
|------|------|------|
| **标题备选** | 约 3 条，每条 12 字以内，强调吸引人、有趣味 | 点击某条选中；不满意点 **重新生成** 只换标题 |
| **简介备选** | 约 2 条，每条 20 字以内，同上 | 点击选中；点 **重新生成** 只换简介 |

**标题和简介都选中后**，底部才会出现导出栏。

## 13. 数据存在哪里？会丢吗？

以下内容都保存在**你这台电脑的浏览器本地**（localStorage），**不会**自动同步到 GitHub 或同事电脑：

- API Key 与模型 ID  
- 已上传的栏目与文稿正文  
- 创作者身份、展开/折叠状态  
- 参考文献历史记录  

| 情况 | 结果 |
|------|------|
| 换电脑 / 换浏览器 | 需重新上传栏目、重新填 Key |
| 清空浏览器缓存 / 无痕模式 | 可能全部丢失 |
| 重装系统 | 可能全部丢失 |

**建议：** 成稿用 Word 或导出文件自行备份；重要参考文稿保留原始文件夹。

## 14. 常见问题

### 打不开网页 / 连接失败

| 检查项 | 处理 |
|--------|------|
| 终端里 `npm run dev` 是否还在跑 | 关了就要重新执行 |
| 地址是否和终端一致 | 有时是 `5174` 等，以终端为准 |
| 是否误用 `https://` | 本地一般是 `http://localhost:5173/` |

### 生成失败 / Failed to fetch

- 确认是用 **`npm run dev` 打开**，不是双击 html  
- 检查 API Key、模型 ID、账户余额  
- 换豆包或 DeepSeek 试一次  

### 上传文件夹后没有栏目

- 确认选的是**父文件夹**（里面包含子文件夹）  
- 子文件夹里要有 `.docx` / `.txt` / `.md`，且文件在**子文件夹第一层**（不要藏在更深子目录）  

### 信源请求没结果或报错

- 确认已 `npm run dev`（需要开发服务器代理学术 API）  
- 检查网络；稍后重试  
- 无结果时可按提示去 Google Scholar、PubMed 手动搜  

### 栏目、Key、文稿突然没了

- 多为清缓存或换了浏览器；需重新上传和配置  

### Mac 提示「无法打开」或权限问题

- Node 安装时若被拦截，到 **系统设置 → 隐私与安全性** 允许

## 17. 名词解释（看不懂命令时翻这里）

| 名词 | 通俗解释 |
|------|----------|
| **终端 / 命令行** | 用文字命令操作电脑的窗口；本文只需要会复制粘贴两三行 |
| **npm** | Node 自带的「软件包管理器」；`npm install` = 安装依赖，`npm run dev` = 启动工具 |
| **localhost** | 表示「本机自己」，不经过公网；只有你能访问 |
| **API Key** | 调用 AI 服务的密码；在厂商网站注册后获得 |
| **模型 ID** | 具体用哪款 AI；不同模型文笔、价格不同 |
| **仓库 / GitHub** | 放项目源代码的网站；本文档所在页面就在仓库的 README |
| **克隆 clone** | 把 GitHub 上的项目复制到你电脑 |
| **代理 / VPN** | 访问国外 AI 接口时，有时需要本机网络工具；豆包、DeepSeek 通常不需要 |

## 18. 给技术同事的附录

### 常用命令

```bash
npm install          # 安装依赖
npm run dev          # 开发模式，默认 http://localhost:5173
npm run dev:clash    # 终端走 HTTP 代理 7890 启动（访问 OpenRouter 等）
npm run build        # 构建到 dist/
npm run preview      # 预览构建结果
```

### 开发代理说明

浏览器不能直接调厂商 API（CORS）。`vite.config.js` 将 `/api/doubao`、`/api/deepseek`、`/api/openrouter` 以及 `/api/openalex`、`/api/crossref`、`/api/wiki-zh` 转发到对应上游。

OpenRouter / 国外模型若 HTTP 500、`ENOTFOUND`，需让 **Node 进程**走代理，例如：

```bash
# 方式 1
HTTPS_PROXY=http://127.0.0.1:7890 npm run dev

# 方式 2：web/.env.development.local
DEV_UPSTREAM_PROXY=http://127.0.0.1:7890
```

### 生产部署注意

- `npm run build` 产物为静态文件；**不会**自带 API 代理。  
- 需在 Nginx / Cloudflare 等配置同样路径反向代理，或改为服务端代理并隐藏 Key。  
- 信源检索依赖 OpenAlex / Crossref / 维基代理，生产环境需一并配置。
